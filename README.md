## Бизнес-цель
Цель данного проекта — разработка высокоточного спам-фильтра для текстовых сообщений. 
Главная техническая особенность: **полный отказ от ML-фреймворков (таких как scikit-learn)**. Вся архитектура, от перевода текста в векторы до вычисления градиентов и метрик, написана с нуля (from scratch) на чистом Python и NumPy.

С точки зрения продукта, основной упор сделан на минимизацию **False Positives (FP)**: алгоритм спроектирован так, чтобы шанс попадания важного пользовательского сообщения в папку "Спам" был минимален.

## Ключевые инженерные и математические решения

Вместо импорта готовых модулей, в проекте реализована собственная математическая база:

* **Векторизованный Градиентный Спуск (Vectorized Gradient Descent):** Отказ от медленных циклов `for` в пользу матричного умножения (`X.T @ error`). Это обеспечило производительность уровня библиотек на C++.
* **Кастомный NLP-модуль (Custom TF-IDF):** Реализован собственный алгоритм Term Frequency - Inverse Document Frequency. Включена защита от взрыва размерности (фильтрация редких токенов) и **L2-нормализация** векторов (`np.linalg.norm`), чтобы длинные тексты не доминировали над короткими.
* **Математика Logistic Regression:** С нуля написаны функции Сигмоиды (с защитой от переполнения `np.clip`) и функция потерь **Cross-Entropy (Log-Loss)**. 
* **Кастомные метрики классификации:** Реализован модуль расчета Confusion Matrix, Precision, Recall и F1-Score на основе булевых масок NumPy.

## Архитектура пайплайна

1. **Text Preprocessing:** Очистка текста с помощью регулярных выражений (`re`), приведение к нижнему регистру, токенизация.
2. **Feature Extraction (`CustomTFIDF`):** 
   * Построение частотного словаря.
   * Расчет IDF с логарифмическим сглаживанием.
   * Трансформация корпуса в разряженную матрицу признаков (матрица `X`).
3. **Modeling (`CustomLogisticRegression`):**
   * Инициализация весов и bias.
   * Обучение модели через Forward Pass и Backward Pass (Gradient Descent).
   * Вывод кривой обучения (Learning Curve) для мониторинга схождения алгоритма.

## Результаты и бизнес-инсайты

На тестовой выборке самописная модель показала индустриальное качество:

* **Precision (Точность спам-фильтра): 99.07%**
* **Recall (Полнота отлова спама): 71.14%**
* **F1-Score: 82.81%**

**Ключевой инсайт (Precision-Recall Tradeoff):** Модель допустила всего **1 False Positive** ошибку на всей тестовой выборке. Это идеальное поведение для спам-фильтра. При необходимости повышения Recall (для более агрессивной фильтрации), в кастомном методе `predict` предусмотрен параметр `threshold` (порог вероятности), который позволяет бизнесу гибко балансировать между ложными срабатываниями и пропуском спама.

В проекте отсутствуют тяжелые ML-зависимости. Нужны только базовые библиотеки для работы с массивами.

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/valanchick/ML-NLP-Framework-from-Scratch-Spam-Classifier.git
   ```
2. Установите зависимости:
   ```bash
   pip install numpy pandas matplotlib
   ```
3. Запустите ноутбук:
   ```bash
   jupyter notebook custom_nlp_classifier.ipynb
   ```
